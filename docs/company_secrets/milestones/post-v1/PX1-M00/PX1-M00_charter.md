# PX1 — Full Industrial Run & Demonstration Proof (durable charter)

**Status:** Established by **PX1-M00** (governance-first charter milestone).

**Boundary:** **PX1** is a **separate post-PV1 governed phase**. It is **not** “Phase VIII” of the closed **v1** arc (**M00–M61**). There is **no** **M62**. Milestone identifiers are **`PX1-MNN`**.

---

## One-sentence charter

**PX1 is a governed post-PV1 execution-and-demonstration phase focused on completing a true full industrial-grade training run on the closed STARLAB substrate and producing bounded, evidence-backed proof that a selected agent can play well enough to win in a governed demonstration, without silently upgrading benchmark, equivalence, ladder, or live-CI claims.**

---

## Why this phase exists

**PV1** delivered a **successful governed research substrate** and a **bounded, honest** campaign outcome (including **`threshold-not-met`** on the frozen PV1 full-run duration semantics). The program still requires, **before considering v2**:

1. A **completed industrial-grade run** under a **PX1** threshold package (schema chartered in **PX1-M00**; values frozen in **PX1-M01**).
2. **Bounded play-quality** evidence and **demo candidate selection** (**PX1-M02**).
3. A **governed demo proof pack** and **winning video** path under explicit acceptance criteria (**PX1-M03**).

**PX1** is **not** a reinterpretation of closed **PV1** results and **not** “cleanup” of PV1.

---

## Success classes (must stay distinct)

1. **Full industrial run success** — run identity + session boundaries + threshold declaration + evidence package + invalidation rules satisfied under PX1.
2. **Play-quality success** — declared criteria on declared surfaces; honest candidate comparison; explicit selected-opponent posture.
3. **Demo / video success** — governed demo contract satisfied (replay/video hooks, opponent/context declaration, bounded “win” semantics).

---

## Phase-level non-claims

PX1 **does not** by itself authorize:

- Ladder claims or public-ranking superiority
- Benchmark universality beyond explicit scope
- Replay↔execution universality
- Live SC2 in CI as default merge gate
- Multi-environment generalization
- **v2** promotion or automatic **v2** readiness

---

## Principles

- Lab-first, benchmark-first, evidence-first  
- Smaller milestones by default  
- Enterprise-grade by default  
- Clean enough to buy (`docs/bicetb.md`)  
- Honest non-claims over marketing language  

---

## Relationship to v2

**v2** remains **outside** the closed **v1** arc until **separately** rechartered. **PX1** is the **pre-v2 bridge** for industrial run + demo proof; finishing PX1 **does not** automatically open **v2**.
