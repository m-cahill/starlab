# Branding and naming

## Canonical project name

**STARLAB** — *Strategic Testing, Analysis, Replay, and Learning Lab*

Use “STARLAB” for public-facing references unless a longer form is needed once.

## Naming rules

- **Repository / package:** lowercase `starlab` for Python package root; repo name may stay `starlab` on the host.
- **Docs:** Title case “STARLAB” in headings; file names use `starlab` or `STARLAB` consistently with existing `docs/starlab.md`.
- **Artifacts:** Prefer a stable prefix (e.g. `starlab-`) for published formats when they exist; define in the milestone that introduces each format.

## Trademark

Perform a reasonable **conflict check** before any public launch, demo site, or commercial adjacent use. This is a diligence hygiene step, not legal advice.

## No placeholder brand drift

Avoid codenames becoming permanent public brands without an explicit decision recorded in `docs/starlab.md`.

## Naming review

Before any **public launch** or **demo site**, record a short naming review note in the closing milestone (owner approval + conflict check status).
